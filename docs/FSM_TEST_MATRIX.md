## 測試 1：狀態機核心流程

| # | 測試項目 | 前置條件 | 操作步驟 | 預期結果 | 實際結果 | 通過 |
|---|---------|---------|---------|---------|---------|-----|
| 1.1 | HUNTING → RETURNING | Bot 運行中 | 手動使用起點卡回城 | 日誌：`HUNTING -> RETURNING` | | |
| 1.2 | RETURNING → TOWN_SUPPLY | 在 RETURNING | 等待到達城鎮 | 日誌：`RETURNING -> TOWN_SUPPLY` | | |
| 1.3 | TOWN_SUPPLY → BACK_TO_FIELD | 在 TOWN_SUPPLY | 等待補給完成 | 日誌：`TOWN_SUPPLY -> BACK_TO_FIELD` | | |
| 1.4 | BACK_TO_FIELD → HUNTING | 在 BACK_TO_FIELD | 等到到達野外 | 日誌：`BACK_TO_FIELD -> HUNTING` | | |
| 1.5 | RETURNING 超時保護 | 在 RETURNING | 90 秒不動 | 強制轉 `TOWN_SUPPLY` | | |
| 1.6 | TOWN_SUPPLY 超時保護 | 在 TOWN_SUPPLY | 5 分鐘不動 | 強制轉 `BACK_TO_FIELD` | | |

## 測試 2：死亡 + 復活流程

| # | 測試項目 | 前置條件 | 操作步驟 | 預期結果 | 實際結果 | 通過 |
|---|---------|---------|---------|---------|---------|-----|
| 2.1 | HP 歸零 → DEAD | HUNTING | 被怪物打死 | 日誌：`HUNTING -> DEAD` | | |
| 2.2 | 自動復活 | DEAD，auto_revive=ON | 等待 1.5s | 點擊復活，日誌：`[DEAD] 嘗試復活` | | |
| 2.3 | 復活成功 | DEAD | HP 恢復 > 0 | 日誌：`DEAD -> IDLE` 或 `DEAD -> HUNTING` | | |
| 2.4 | 復活超時 | DEAD，auto_revive=ON | 等待 60 秒 | 強制回 IDLE，日誌：`超時 60s` | | |
| 2.5 | 手動復活關閉 | DEAD，auto_revive=OFF | 等待 | 日誌：`auto_revive 關閉，等待手動復活` | | |
| 2.6 | 復活失敗超次 | DEAD | 20 次點擊失敗 | 強制回 IDLE | | |
